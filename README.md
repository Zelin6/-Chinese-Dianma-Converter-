# -Chinese-Dianma-Converter-
一个用于中文电码（汉字电报码）与中文字符相互转换的C++命令行工具。支持数字电码（四位数码）和字母电码（三位字母码）之间的转换。
功能特性
🔄 双向转换：支持中文↔数字电码、字母电码↔中文的转换

📁 多数据源：支持从文件加载完整电码数据库，也提供内置基础数据

🖥️ 双模式：交互式命令行界面和直接命令行参数两种使用方式

🔍 智能识别：自动识别输入类型（中文、数字电码或字母电码）

💾 UTF-8支持：正确处理UTF-8编码的中文字符

🪟 跨平台：支持Windows、Linux和macOS系统

编译与运行
环境要求
C++11兼容的编译器（g++、clang++、MSVC等）

支持UTF-8编码的终端

编译方法
bash
# 使用g++编译
g++ -std=c++11 -o dianma_converter main.cpp

# 或者使用cmake
mkdir build && cd build
cmake ..
make
运行程序
bash
# 交互模式
./dianma_converter

# 从文件加载数据
./dianma_converter -f dianma_data.txt

# 中文转电码（命令行模式）
./dianma_converter -c "中文文本"

# 数字电码转中文
./dianma_converter -d "0001 0002 0003"

# 字母电码转中文
./dianma_converter -l "AAB AAC AAD"

text
数字电码  字母电码  中文  名称  备注
0001      AAB       一    ...
0002      AAC       丁    ...
0003      AAD       七    ...

text
main.cpp                 # 主程序文件
DianmaConverter类        # 核心转换类
  ├── 数据加载模块        # 从文件或内置数据加载映射
  ├── 转换功能模块        # 各种转换方法实现

注意事项
数据完整性：内置数据仅包含少量示例字符，建议提供完整的电码数据文件以获得完整功能

编码问题：确保终端和输入文件使用UTF-8编码

电码格式：数字电码为4位数字，字母电码为3位大写字母

Windows系统：程序已包含控制台编码设置，支持Windows的中文显示
